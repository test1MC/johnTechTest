Scenario Builder

A streamlined workflow to build Scenarios including multi-month rate rules, rate environment responses and adjustment rules.

 

  Scenario Builder Step Description
  1) Name Scenario Set a Scenario name, description, and duration
  2) Set My Strategy Set a bank's rate to test, and then set any teaser duration, as well as any advertising schedule or other non-rate factors
  3) Set Rate Environment Set a forecast for the macro rate environment, and then set your bank and competitors responses to these rate changes, as well as competitor changes not triggered by a rate environment change.
  4) Set Adjustment Rules

Set rules to override modeled projections based on expert judgment.
  5) Review and Run Scenario Review the Scenario parameters before running.

 

There is 18-minute demo that details how to create a new scenario from scratch and run it. Click here to access the How to Create and Run a Scenario eLearning.

Key Terms: Beta and Lag

Lag: The gap in time between a macro change and a product rate change

  • For example, if the Fed Funds rate increases in December and the lag is set to 1, the response is effective in January
  • Lags can be set to 0, 1 or 2 months

Beta: The percent of each macro change passed through to the product rate

  • For example, if the Fed Funds rate goes up by 25 bp and the beta is set to 0.8, the rate would increase by 25 * 0.8 = 20 bp
  • Enter a 60% beta as “0.60”

Set Adjustment Rules

Choose from several overlays that adjust your Scenario predictions based on your expert judgment and knowledge of your portfolio, sales force, competitors, and environment. The objective of these rules is to allow you to account for influencers that will impact the portfolio but are unlikely to be fully captured in the elasticity model.

Key Adjustment Rule Applications

  • Only one “standard” Adjustment Rule can apply to a single cell in the same month- the bottom rule in the inventory applies.
    • Switch Allocation and Change Caps rules are considered “non-standard” and can be applied along with a "standard" rule.
    • Between Static Assumption, Dynamic Assumption, and Apply Adjacent Model rules though- only one can apply in a single month for a single cell.
  • Change Caps rules can apply at the same time as Dynamic Assumption and Apply Adjacent Model rules, but cannot apply at the same time as Static Assumption rules.
    • If a Change Caps and Dynamic Assumption rule apply in the same month, cap references are based on the original curve inputs (e.g., if the curve is shifted up 20% but extrapolation is capped at 0% above the highest historical data point, the cap is not shifted by the +20% Dynamic Assumption rule.
    • If there is no Change Caps rule in the same month, the default extrapolation caps are then shifted by the Value in the Dynamic Assumption rule (e.g., if the curve is shifted up 20%, the caps are also be shifted up 20%).
  • Order of Application:
 
1.    Apply the lowest adjustment rule in the inventory in this cell for this month (with the exception of Change Caps and Switch Allocation rules).
2.    If the active rule is not static and there is a Change Caps rule: Apply it.

If the active rule is not static and there is no Change Caps rule: Apply the default extrapolation.

3.    No matter what rule applies: Apply the global caps (e.g., 0% to 100% for Percent Attrition).
Setting a Static Assumption Adjustment Rule

 

Static Adjustment Rules override the selected models with a single fixed value – either an average of recent months, or an actual value (e.g., you might want to use the last 3-month average in certain cells instead of the models if they have poor fit and low sample size).

Use Cases:

  • You don’t trust this curve. The fit isn’t great, particularly recently and you want to use the last 3 month average instead.

  • You don't believe that Attrition would never go as low as the curve indicates and would trust the projections more if a static assumption was used instead.

  • Despite the curve being fairly flat, recent performance has been a lot higher and you believe your institution will be able to maintain that higher level of performance.

Set the parameters for a static override of the current model with a selected value.
1.    Click the Select button under Static Assumption.

Tip: All of these selections have been pre-defined as part of your deployment process and are specific to your bank.

6.    Click the Static Assumption Value drop-down and choose 3 Month Average, 6 Month Average, 12 Month Average or Enter a Value.

Setting a Dynamic Assumption Adjustment Rule

 

Dynamic Adjustment Rules shift the curve while maintaining its slope. To shift the curve up or down, all projections are multiplied by a specified value. For example, you can apply a 10% haircut to money market acquisition after month 3 to account for promotional fatigue, or a 5% increase in acquisition following a change in sales force incentives.

Use Cases:

  • You think the CD market is becoming saturated and want to shift CD acquisition curves down 10% at every relative rate.

  • You expect promotional decay if you keep these rates active for 3 full months and therefore - want to shift the projections down by 20% in the third month.
